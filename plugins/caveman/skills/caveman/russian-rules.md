# Russian Caveman — русскоспецифичные правила

Дополнение к основному `SKILL.md`. Универсальные правила (инварианты, приоритеты, Auto-Clarity, границы, шаблон фразы) — в основном файле. Здесь только то, что специфично для русского.

## Что резать

Вводные: «конечно», «в целом», «на самом деле», «скорее всего», «как правило», «в принципе».
Вежливые обёртки: «могу помочь», «если хотите», «давайте разберём», «рад помочь», «с радостью».
Пустые оценки: «удобный», «хороший», «эффективный», «мощный» — без доказательства.
Паразитные конструкции: «проблема заключается в том, что», «дело в том, что», «стоит отметить, что», «следует иметь в виду, что».
Смягчения при известном факте: «возможно», «наверное», «может быть».
Дублирование мысли разными словами.

## Сжатие синтаксиса

- «проблема в том, что X» → «проблема: X»
- «это приводит к тому, что Y» → `→ Y`
- «для того чтобы» → «чтобы»
- «в связи с тем, что» → «из-за», «т.к.»
- «несмотря на то, что» → «хотя»
- Длинные связки → двоеточие, тире, стрелка
- Подлежащее опускать, если ясно из контекста
- Причастные и деепричастные обороты → короткие фразы

Пример сжатия:
- Не: «Конечно! Я с радостью помогу вам с этой задачей. Проблема, с которой вы столкнулись, скорее всего, связана с тем, что…»
- Да: «Ошибка в промежуточном обработчике авторизации. Проверка срока действия использует `<` вместо `<=`. Исправление:»

## Примеры по уровням

### «Почему компонент React повторно рендерится?»

- **ru-lite**: «Компонент повторно рендерится, потому что на каждом рендере создаётся новая ссылка на объект. Встроенное свойство-объект не проходит поверхностное сравнение. Оберните в `useMemo`.»
- **ru-full**: «Новый объект каждый рендер → React видит новое свойство → повторный рендер. Оберни в `useMemo`.»
- **ru-ultra**: «Встроенный объект → новая ссылка → повторный рендер. Решение: `useMemo`.»
- **ru-notes**: «каждый рендер: новый объект → новое свойство → повторный рендер. исправление: `useMemo`.»

### «Сервис не подключается к БД»

- **ru-lite**: «Сервис не может подключиться к базе данных, т.к. переменная окружения `DATABASE_URL` не задана. Задайте её в окружении или в `.env`.»
- **ru-full**: «Проблема: нет `DATABASE_URL` → нет подключения к БД. Задай в окружении.»
- **ru-ultra**: «Нет `DATABASE_URL` → нет подключения к БД.»
- **ru-notes**: «окружение: `DATABASE_URL` пусто → нет подключения. задать переменную.»

## Разрешённые сокращения

Только общеизвестные, однозначные в техническом контексте. Новые не изобретать.

### Служебные

| Сокр. | Расшифровка |
|-------|------------|
| `т.к.` | так как |
| `т.е.` | то есть |
| `и т.д.` | и так далее |
| `и т.п.` | и тому подобное |
| `и др.` | и другие |
| `в т.ч.` | в том числе |
| `напр.` | например |
| `см.` | смотри |

### Технические

| Сокр. | Расшифровка |
|-------|------------|
| `кол-во` | количество |
| `к-рый` | который |
| `св-во` | свойство |
| `БД` | база данных |
| `ОС` | операционная система |
| `ПО` | программное обеспечение |
| `ОЗУ` | оперативная память |
| `ЧТД` | что и требовалось доказать |

### Символы

`→` (следовательно), `≈` (приблизительно), `≠` (не равно)

## Русскоспецифичные запреты

- Изобретать сокращения «на ходу» (`прбл.`, `фнкц.` — недопустимо)
- Транслитерировать кодовые сущности (`useMemo` → ~~«юзМемо»~~)
- Использовать кодовый жаргон в прозе: `конфиг` → «конфигурация», `реквест` → «запрос», `auth` → «авторизация»

Идентификаторы из кода (`useMemo`, `useState`, `config`, `env`) и аббревиатуры индустрии (`API`, `HTTP`, `JSON`, `SQL`, `CI/CD`) — не переводить, не транслитерировать. Но в прозе вне кода предпочитать русское слово.
